OpenShift Express Application Wizard
New Wizard We developed a wizard that would allow you to import an OpenShift Express application to your Eclipse workspace. The wizard holds all operations involved in creating a new application on OpenShift Express.

A frist step would require you to supply your credentials. A link would open the signup page if you have no account on OpenShift Express yet.

You may eventually have to create a domain first.

The wizard allows you to create a new application.

It then lists your application.

A dialog lists the details for your new application.

The last step involved determines to what local location the git repository is cloned to. You may also tell our tooling to create a server adapter that will allow you to publish changes to the OpenShift Express PaaS.

The wizard will clone the repo...

...import the OpenShift Express application to your workspace...

...and create the Eclipse WTP server adapter for you.

OpenShift Express REST client
REST Client We also developed a REST client in Java for OpenShift Express. It pretty much offers all features that are currently offered by the rhc-* command line tools. Even though it's pretty new, it's code base is pretty well tested since it's covered well by unit-tests and used in our application wizard and in the Forge OpenShift Express plugin. The plugin is currently still an eclipse plugin, but we will strip its Eclipse dependencies and move it to github at some point. You may check the current code in our SVN.

JSF @FacesConverter validation

Eclipse JSF Tools (WTP) has an annoying issue with faces converter validation. It does not support faces converters annotated @FacesConverter. So if your project uses such converters in <f:converter converterId="..."/> you will get a warning. But JBoss Tools 3.3.0.M5 has a new quick fix which will help you to disable Facelet Validator to get rid of that warning.

Configuring Problem Severity via Quick Fix

In case of some JBoss Tools validator (CDI, JSF, Seam) reports a false problem, you can disable the corresponding validation rule in Preferences via the new quick fix "Configure Problem Severity".

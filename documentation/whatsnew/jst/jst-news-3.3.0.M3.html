JSF 2

@ManagedBean support

JSF Tools now support managed beans annotated @ManagedBean. This support includes EL code completion, validation, refactoring, OpenOns (Ctrl+Click) and searching (Ctrl+Shift+G).

Related Jira

<faces-config metadata-complete="true">

If <faces-config metadata-complete="true"> then JSF Tools will ignore bean annotated @ManagedBean according to JSF-2 spec.

JSF capabilities

Projects w/ no web.xml

It was a pretty annoying bug in Add JSF Capability dialog which required web.xml. That obsolete dialog has been removed. Now all you need to enable JBoss JSF Tools support is to install JSF facet into the project. So Project->Configuration->Add JSF Capability will lead you to Project Properties->Facets property page. Web.xml is not required there.

Related Jira

If some of your JSF project depends on another Java Project then both project should have JBoss Tools Knowledge Base builder installed to provide code completion, validation and other features for all resources from both projects. If for some reason not all of your dependent projects configured correctly then JBoss Tools will add a warning to the project. A QuickFix is available for such a warning. And of course you can disable that warning in Preferences.

Configuring a Session Factory

In order to use the Hibernate Console, you will need to create/configure a session factory. This is done by using the new (or configure) button from the Session Factory's Window toolbar.
The configuration dialog will pop up, and you will need to fill in:

Display Name
Name under which the session factory will be identified in the system. Can be choosen freely.
Mapping Locations & Files
Hibernate mapping files or directories under which these files are located. Directories will be scanned recursively for mapping files (by default files ending with .hbm.xml or .hbm). Mapping file extensions can be configured in the options dialog (Tools | Options | Hibernate)
Classpath Entries
JAR files and directories defining your application's classpath. The classpath typically includes your mapped objects (with dependencies) and the JDBC driver.
Configuration tab
Database related information. Includes the hibernate dialect, the database URL, user and password.
Advanced tab
Contains advanced hibernate properties. They should be enterd in a properties like syntax (the same as in hibernate.properties).
BeanShell Imports tab
Manages java imports used by the BeanShell Editor
